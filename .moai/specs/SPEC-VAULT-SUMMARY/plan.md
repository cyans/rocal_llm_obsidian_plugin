# SPEC-VAULT-SUMMARY: 구현 계획

## 1. 기술 아키텍처

### 1.1 컴포넌트 구성도

```
┌─────────────────────────────────────────────────────────────┐
│                    ChatView (UI)                            │
└─────────┬───────────────────────────────────────────────────┘
          ▼
┌─────────────────────────────────────────────────────────────┐
│                    ToolRegistry                             │
│  ┌──────────────────┐  ┌──────────────────┐                │
│  │VaultReadContents │  │VaultSummarizeTool│                │
│  └────────┬─────────┘  └────────┬─────────┘                │
│           │                     │                           │
│  ┌────────▼─────────┐  ┌───────▼──────────┐                │
│  │OutlineOrganizer  │  │VaultPreprocessor │                │
│  └──────────────────┘  └────────┬─────────┘                │
└──────────────────────────────────┼──────────────────────────┘
                                   │
          ┌────────────────────────▼────────────────────────┐
          │     API-first + CLI-assisted pre-processing     │
          │  (search/outline/links/backlinks/read optional) │
          └───────────────┬─────────────────────────────────┘
                          │
          ┌───────────────┴──────────────────────┐
          │      Obsidian API + Optional CLI     │
          └───────────────┬───────────────────┘
                          │
          ┌───────────────┴───────────────────┐
          │         Vault Files                │
          └────────────────────────────────────┘
```

### 1.2 데이터 흐름

```
1. 사용자 요청
   사용자 → ChatView → AgentController
                        ↓
2. 대상 파일 결정
   활성 파일 / 명시 경로 / 검색 결과
                        ↓
3. 프리프로세싱 (선택)
   CLI search / outline / links / backlinks
   또는 API 기반 직접 읽기
                        ↓
4. 본문 확보
   필요한 소수 파일만 API 또는 CLI read
                        ↓
5. 요약/재구성
   VaultSummarizeTool / OutlineOrganizerTool → LLMService
                        ↓
6. 결과 표시/저장
   ChatView 표시 → 필요 시 write_to_file
```

## 2. 구현 단계별 작업

### Phase 1: API 기반 MVP

**파일:**
- `src/tools/VaultReadContentsTool.ts`
- `src/tools/VaultSummarizeTool.ts`
- `src/ui/ChatView.ts`
- `src/agent/ToolRegistry.ts`

**작업:**
1. 활성 파일/명시 파일 읽기
2. 단일 파일 요약 도구 구현
3. 채팅 응답 포맷 정리

**인수 기준:**
- 활성 파일 요약 가능
- 없는 파일/빈 파일 오류 처리
- 채팅에서 바로 결과 확인 가능

### Phase 2: 다중 파일 요약

**작업:**
1. 다중 파일 입력 처리
2. 파일별 요약 + 통합 요약
3. 처리 대상 상한선 적용

**인수 기준:**
- 최대 10개 파일까지 처리 가능
- 출처 파일 표시
- 통합 요약 반환

### Phase 3: 목차 기반 정리

**작업:**
1. outline 입력 파싱
2. 섹션별 재구성
3. 새 노트 저장 연동

**인수 기준:**
- 목차 기준 정리 가능
- 섹션별 출처 표시
- 저장 연동 가능

### Phase 4: CLI-assisted optimization

**작업:**
1. CLI 감지 래퍼 구현
2. `search`, `outline`, `links`, `backlinks` 프리프로세싱 추가
3. `read`의 제한적 사용
4. API fallback 및 상태 노출

**인수 기준:**
- CLI가 있으면 후보 축소 또는 구조 추출에 활용
- CLI 실패 시 API 경로 유지
- 디버그 상태에 사용 경로 표시

## 4. 리스크 및 완화 전략

### 4.1 기술 리스크

| 리스크 | 영향 | 확률 | 완화 전략 |
