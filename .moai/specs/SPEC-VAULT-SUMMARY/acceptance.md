# SPEC-VAULT-SUMMARY: 인수 기준

## 1. Given/When/Then 시나리오

### Scenario 1: 단일 파일 요약

**Given** 사용자가 볼트 내 "Project Notes.md" 파일을 열어 둔 상태에서
**When** 채팅에 "현재 파일 요약해줘"라고 입력하면
**Then** 다음이 발생해야 한다:
- 파일 내용이 읽혀진다
- LLM이 내용을 요약한다
- 요약 결과가 채팅에 표시된다
- 요약 결과를 복사할 수 있다

### Scenario 2: 다중 파일 배치 요약

**Given** 사용자가 5개의 파일 경로 또는 검색 결과를 지정하고
**When** 채팅에 통합 요약을 요청하면
**Then** 다음이 발생해야 한다:
- 각 파일이 순차적으로 처리된다
- 진행 상황이 표시된다 (1/5, 2/5, ...)
- 각 파일의 개별 요약이 생성된다
- 통합 요약이 생성된다
- 결과가 섹션별로 정리되어 표시된다

### Scenario 3: 목차 기반 정리

**Given** 사용자가 다음 목차를 입력하고:
```
1. 서론
2. 본론
3. 결론
```
**And** 3개의 관련 파일을 선택하고
**When** 채팅에서 목차 기준 정리를 요청하면
**Then** 다음이 발생해야 한다:
- 각 파일 내용이 분석된다
- 목차 섹션별로 관련 내용이 분류된다
- 각 섹션에 출처 파일이 표시된다
- 재구성된 내용이 표시된다

### Scenario 4: CLI-assisted pre-processing

**Given** Obsidian CLI가 설치되어 있고 관련 파일이 많은 요청이며
**When** 시스템이 사전 축약을 수행하면
**Then** 다음이 발생해야 한다:
- 시스템이 `search`, `outline`, `links`, `backlinks` 중 하나 이상을 먼저 사용할 수 있다
- 전체 본문 대신 축약된 입력으로 후보를 줄인다
- 최종 요약 품질에 필요한 파일만 본문 읽기로 이어진다

### Scenario 5: CLI fallback

**Given** Obsidian CLI가 설치되어 있지 않거나 실행에 실패하고
**When** 동일한 요청을 처리하면
**Then** 다음이 발생해야 한다:
- 시스템이 자동으로 Obsidian API를 사용한다
- 기능이 정상적으로 작동한다
- 디버그 상태에서는 API fallback 여부를 확인할 수 있다

### Scenario 6: 대용량 파일 처리

**Given** 사용자가 1MB 크기의 파일을 선택하고
**When** 요약을 요청하면
**Then** 다음이 발생해야 한다:
- 파일이 청크 단위로 처리된다
- 진행 상황이 표시된다
- 요약이 완료되면 결과가 표시된다

## 2. 엣지 케이스 테스트

### Edge Case 1: 없는 파일
- **조건**: 존재하지 않는 파일 경로 입력
- **예상**: "파일을 찾을 수 없습니다" 에러 메시지
- **복구**: 올바른 파일 경로 또는 활성 파일 확인 안내

### Edge Case 2: 빈 파일
- **조건**: 0KB 파일 선택
- **예상**: "파일이 비어있습니다" 메시지
- **복구**: 다른 파일 선택 안내

### Edge Case 3: CLI 사용 불가
- **조건**: CLI 미설치, 실행 실패, 권한 문제
- **예상**: API fallback
- **복구**: 요약 기능은 계속 사용 가능

### Edge Case 4: LLM 타임아웃
- **조건**: 요약 요청이 30초 이상 소요
- **예상**: "요약 시간 초과" 메시지
- **복구**: 재시도 또는 간단 요약 옵션

### Edge Case 5: 너무 긴 요약
- **조건**: 원본이 요약보다 긴 경우
- **예상**: "요약에 실패했습니다" 메시지
- **복구**: 다른 요약 스타일 시도

### Edge Case 6: 특수 문자 포함
- **조건**: 파일명에 특수字符 (?, *, ", <, >, |)
- **예상**: 특수字符 처리 또는 에러
- **복구**: 안전한 에러 메시지
